# Design Problem

We now know that a digital sound is a collection of samples.

But where do those samples come from?

Suppose we want to create a tone.

Should we type forty-four thousand numbers by hand?

Of course not.

Instead, we describe a mathematical rule.

The computer repeatedly applies that rule to generate every sample.

The resulting pattern of values is called a **waveform**.

---

# Why This Matters

Every synthesized sound begins with a waveform.

Examples include:

- musical notes
- warning beeps
- engine sounds
- laser blasts
- user interface sounds
- ambient drones

Different rules produce different waveforms.

Different waveforms produce different sounds.

---

# Mental Model

Imagine drawing a path across a graph.

```text
Time

↓

Rule

↓

Value
```

Every moment in time asks exactly the same question.

> **What value should this sample have?**

The waveform is simply the rule that answers that question.

---

# Representation

Suppose we generate one sample.

```text
Time

↓

0.00

↓

Sample = 1.00
```

A moment later...

```text
Time

↓

0.01

↓

Sample = 0.99
```

Then...

```text
Time

↓

0.02

↓

Sample = 0.96
```

Repeating this computation thousands of times produces an entire waveform.

The sound emerges from the pattern.

---

# Starter Implementation

One simple waveform is a cosine.

```python
waveform = [
    math.cos(
        2 * math.pi * frequency * n / sample_rate
    )
    for n in range(num_samples)
]
```

Notice something important.

We are not generating one sound.

We are generating one sample at a time.

The computer repeats the same rule for every moment in time.

---

# Dissecting the Process

Waveform generation follows the same computational pattern we've already seen throughout this unit.

```text
Generate Representation

↓

Repeat

↓

Transform Time

↓

Produce Sample

↓

Repeat Again
```

The representation grows one sample at a time until the sound is complete.

---

# Design Principle

## Sounds emerge from repeated rules.

The waveform is not stored as a picture.

It is not stored as music.

It is generated by repeatedly applying one mathematical relationship across time.

Simple rules often produce surprisingly rich sounds.

# Practical Observation

Although many different waveforms exist...

...they all share the same computational structure.

For every moment in time:

1. Determine the current time.
2. Apply a mathematical rule.
3. Produce one sample.
4. Repeat.

The mathematics changes.

The computational process remains remarkably consistent.

# Common Misconceptions

### "A waveform is a picture."

Not exactly.

Graphs are visualizations of waveforms.

The waveform itself is the sequence of sample values.

---

### "Computers store tones."

No.

Computers generate or store samples.

Our ears interpret the resulting waveform as a tone.

---

### "Every sound requires recording."

Not at all.

Many sounds are synthesized entirely through computation.

The waveform itself is created algorithmically.

---

### "Waveforms are only used in music."

No.

Waveforms appear throughout computing.

Games use them for:

- interface sounds
- procedural effects
- ambient audio
- synthesizers
- simulation

They are another computational representation.
